# 📁 Database

This folder contains the centralised SQLite database used across the entire **RLIS — Revenue Leakage Intelligence System** pipeline.

---

## 📄 File

| File | Description |
|---|---|
| `revenue_leakage.db` | Shared SQLite database storing all analytical output tables generated by Notebooks 01–03 and consumed by Notebook 04 |

---

## 🗂️ Tables

| Table | Created By | Description |
|---|---|---|
| `superstore_analysis` | Notebook 01 | Enriched Superstore dataset with Discount Abuse and Pricing Leakage flags and amounts per transaction |
| `olist_billing_analysis` | Notebook 02 | Merged Olist orders-items-payments table with payment gap calculation and Billing Leakage flags |
| `uci_customer_return_analysis` | Notebook 03 | Customer-level return aggregation with Return Rate % and Return Risk Category (Low / Medium / High) |
| `master_kpi_summary` | Notebook 04 | Consolidated 4-row KPI summary with total leakage per type, Contribution %, Severity, and Recovery Category |

---

## ⚙️ Usage

The database is created automatically when **Notebook 01** is executed for the first time.  
Each subsequent notebook appends its output table to the same database file.

> **Prerequisite:** Run notebooks in order — `01 → 02 → 03 → 04` — to ensure all tables exist before the Master consolidation step.

---

## ⚠️ Note

`revenue_leakage.db` is **not tracked by Git** (listed in `.gitignore`).  
To reproduce it locally, execute all four notebooks sequentially from the project root.
